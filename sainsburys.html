Project Objective

Developed a self-scan interface for Sainsbury to enhance the shopping experience by allowing customers to quickly and easily scan and pay for their items without cashier assistance, from errors occurring from weighted items. In addition, reduce the space for each self-scan.

Content Design

Objective: Ensure that the self-scan interface was intuitive, with clear instructions and minimal text to facilitate a seamless self-checkout process.

Approach:
  • Created concise, easy-to-understand instructional text and on-screen prompts to guide users through the scanning and payment process.
  • Used simple and direct language to reduce cognitive load and prevent user errors.
  • Included visual cues and icons to complement text instructions, ensuring clarity for users with different language proficiencies.

Outcome: Developed a content strategy that prioritized user-friendly language and clear visual instructions, leading to a more intuitive and efficient self-scan experience.

Wireframing

Objective: Visualize the layout and flow of the interface to ensure intuitive navigation and ease of use.

Approach:
  • Created low-fidelity wireframes using Figma to outline the basic structure and key elements of the interface.
  • Focused on the logical arrangement of functions such as item scanning, cart review, payment options, and receipt printing.

Outcome: Produced a set of wireframes that provided a clear blueprint for the self-scan interface, ensuring a logical flow and easy navigation for users.

Visual Design

Objective: Develop a visually appealing and brand-consistent interface that enhances the user experience.

Approach:
  • Used Figma to create high-fidelity mock-ups, incorporating Sainsburys branding elements such as colour scheme, typography, and logos.
  • Focused on a clean, modern design with ample white space to prevent clutter and ensure readability.
  • Designed intuitive icons and buttons to facilitate easy interaction, with consistent use of colour to denote different actions (e.g., green for confirming actions, red for cancellations).
  • Ensured visual accessibility by adhering to contrast ratio guidelines and using legible fonts and sizes.

Outcome: Delivered a polished visual design that not only adhered to Sainsburys brand guidelines but also enhanced the overall usability and aesthetic appeal of the self-scan.

Prototyping

Objective: Create an interactive prototype to simulate the user experience.

Approach:
  • Developed an interactive prototype using Figma, incorporating all elements from the visual design phase.
  • Enabled key interactions such as scanning items, reviewing the cart, selecting payment methods, and printing receipts to simulate the complete self-scan process.

Outcome: Produced a functional prototype that closely mimicked the final product.
