UX Conference Project

Through my experience during School of UX, I was able to develop further knowledge UX/UI knowledge which I applied to real-life scenarios. The below presentation details the key design process which focused on real project focus on event management Dasboard.

Background

During the last part of the course, it bought together the different elements of what I had learnt during the course to the final project. The final project involves a real client, with requirements and desire outcome, in this case events dashboard. For me, it allowed me to complete a project, by going through each step, implementing the key learning all consolidated in the final project. There were key considerations such as accessibility, and use flow, which I believe need to be consider before even design a solution or MVP. I was able to double down, on the content layout, wireframing, building out the style guide and UI components. The process of prototyping, allowed me to keep experimenting with variances and variables to produce a functional app. The project demonstrate clear progress from the previous projects, continuous adding layers to skills and knowledge.

Project Objective

Developed an event sales dashboard for UXConference to help organizers track ticket sales, manage attendee information, and generate sales reports. The goal was to create a user-friendly and intuitive interface that provides valuable insights and streamlines the sales process.

Business Research

Objective: Understand the needs of UXConference and its users to ensure the dashboard meets both business objectives and user expectations.

Approach:
  • Conducted stakeholder interviews to gather insights into business goals, current sales tracking methods, and areas needing improvement.

Outcome: Developed a comprehensive understanding of business requirements and user needs, providing a solid foundation for the dashboard design.

User Flow

Objective: Map out the steps users take to interact with the dashboard, ensuring a smooth and intuitive experience.

Approach:
  • Developed user journey maps to visualize the steps users take to complete key tasks such as tracking sales, generating reports, and managing attendee ticket information.
  • Identified key touchpoints and potential pain points, ensuring the design addressed these areas for a seamless experience.

Outcome: Provided a comprehensive user flow that guided the design process, ensuring all user needs and expectations were met.

Information Architecture

Objective: Organize the information within the dashboard to ensure intuitive navigation and ease of use.

Approach:
  • Created a site map to visualize the structure and hierarchy of the dashboard’s content and features.
  • Ensured that frequently accessed features such as sales tracking, report generation, and tickets sales information were easily accessible from the main dashboard

Outcome: Developed a clear and logical information architecture that facilitated easy navigation and access to key features.

Content Design

Objective: Create clear, concise, and engaging content to guide users through the dashboard and improve overall usability.

Approach:
  • Simplified navigation labels and instructions to ensure clarity and reduce cognitive load.
  • Created content templates for consistent messaging across different sections of the dashboard, ensuring a cohesive user experience.

Outcome: Produced user-friendly content that enhanced navigation, improved comprehension, and provided a more intuitive dashboard experience.

Wireframing

Objective: Visualize the structure and layout of the dashboard to ensure intuitive navigation and functionality.

Approach
  • Created low-fidelity wireframes using Figma to outline the dashboard’s basic structure and key elements.
  • Focused on essential screens such as the main dashboard, order, revenue, analytics and messaging.

Outcome: Developed comprehensive wireframes that served as a blueprint for the dashboard design, ensuring a user-centered layout and intuitive navigation.

Visual Design

Objective: Develop a visually appealing and brand-consistent interface that enhances the user experience.

Approach:
  • Used Figma to create high-fidelity mockups, incorporating UX Conference’s branding elements such as color scheme, typography, and logos.
  • Focused on a clean, modern design with intuitive icons and buttons to facilitate easy interaction.
  • Ensured consistency in visual design across all screens, maintaining a cohesive look and feel.
  • Applied visual hierarchy principles to guide user attention to important elements and actions, such as sales trends and key metrics.

Outcome: Delivered a polished visual design that enhanced usability and aligned with the brand’s identity, providing an aesthetically pleasing and effective user experience.

Accessibility

Objective: Ensure the dashboard is accessible to all users, including those with disabilities.

Approach:
  • Adhered to WCAG 2.1 guidelines throughout the design process, ensuring compliance with accessibility standards.

    • Outcome: Delivered an inclusive dashboard design that meets accessibility standards, ensuring a positive experience for all users.

    Prototyping

    Objective: Create an interactive prototype to simulate the user experience and gather feedback for further refinement.

    Approach:
  • Developed an interactive prototype using Figma, incorporating all elements from the visual design phase.
  • Enabled key interactions such as sales tracking, revenue, and attendee information, analytics to simulate the complete dashboard experience.

    • Outcome: Produced a functional prototype that closely mimicked the final intended product, providing valuable insights for further refinements and ensuring a user-centric design.
